About

About ♠

The aim of this project is to establish a GNU/Linux web server with automated deployment of websites. This process entails establishing a Virtual Private Server (VPS), implementing fundamental security measures, setting up the web server, and automating deployment: whenever a commit is made to the git repository, the website gets updated automatically.

We will utilize a Public Domain license

Major technical solutions compared

This project will compare different web servers (Nginx vs Apache) and Ubuntu and Debian and deployment automation methods to determine the most efficient and reliable setup. We will compare it based on ease of configuration, security features, uptime and maybe more.

Weekly tasks ☆

Week 1:

  1. Research Web Servers (Compare Nginx vs Apache)
  2. Research Operating Systems (Compare Ubuntu vs Debian or other OS)
  3. Research Deployment automation methods
  4. Research a VPS Provider
  5. Purchase a VPS (whichever was researched to be best)
  6. Create GitHub repository

Week 2:

  1. Install Web Server (whichever was researched to be best)
  2. Set up SSH Keys for each of us
  3. Set up Firewall
  4. Disable root login
  5. Set timezone and time
  6. Install essential packages + updating

Week 3:

  1. Configure deployment scripts to automatically update web servers using GitHub
  2. set up cron jobs to automate deployments on the VPS
  3. test the automated deployments by using the git commands
